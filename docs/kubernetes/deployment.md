# Deployment

## Terraform / OpenTofu

- Terraform official: <https://www.terraform.io/>
- Terraform GitHub: <https://github.com/hashicorp/terraform>
- OpenTofu official: <https://opentofu.org/>
- OpenTofu GitHub: <https://github.com/opentofu/opentofu>

Infrastructure-as-Code tools that provision Kubernetes clusters and resources across clouds. OpenTofu is the Linux Foundation fork of Terraform.

## Crossplane

- Official: <https://www.crossplane.io/>
- GitHub: <https://github.com/crossplane/crossplane>

CNCF project that turns Kubernetes into a universal control plane, managing cloud infrastructure and services through Kubernetes CRDs.

## Pulumi

- Official: <https://www.pulumi.com/>
- GitHub: <https://github.com/pulumi/pulumi>

Infrastructure-as-Code platform that lets you define cloud and Kubernetes resources using general-purpose languages (TypeScript, Python, Go, .NET, Java).

## Cluster API

- Official: <https://cluster-api.sigs.k8s.io/>
- GitHub: <https://github.com/kubernetes-sigs/cluster-api>

Kubernetes SIG project providing declarative, Kubernetes-style APIs for cluster creation, configuration, and management across many providers.

## Google Kubernetes Config Connector

- Official: <https://cloud.google.com/config-connector/docs/overview>
- GitHub: <https://github.com/GoogleCloudPlatform/k8s-config-connector>

Google Cloud's Kubernetes addon that exposes GCP resources as Kubernetes CRDs, enabling GitOps-style management of Google Cloud infrastructure.

## AWS Controllers for Kubernetes (ACK)

- Official: <https://aws-controllers-k8s.github.io/community/>
- GitHub: <https://github.com/aws-controllers-k8s/community>

A set of Kubernetes controllers that let you create and manage AWS service resources directly from Kubernetes.

## Kordent (KCM)

- Official: <https://kordent.io/>
- GitHub: <https://github.com/k0rdent/kcm>

Mirantis-backed open source project (Kubernetes Cluster Manager) for managing the lifecycle of clusters and platform services across providers.
