# Installers

## Kops

- Official: <https://kops.sigs.k8s.io/>
- GitHub: <https://github.com/kubernetes/kops>

Kubernetes Operations: a production-grade tool for installing, upgrading, and managing self-hosted Kubernetes clusters, primarily on AWS and GCP.

## Kubeadm

- Official: <https://kubernetes.io/docs/reference/setup-tools/kubeadm/>
- GitHub: <https://github.com/kubernetes/kubeadm>

Official Kubernetes tool that bootstraps minimum-viable, conformant clusters following best practices, designed to be used as a building block.

## Kubespray

- Official: <https://kubespray.io/>
- GitHub: <https://github.com/kubernetes-sigs/kubespray>

Ansible-based installer for deploying production-ready Kubernetes clusters on bare metal, VMs, and major cloud providers.

## Minikube

- Official: <https://minikube.sigs.k8s.io/>
- GitHub: <https://github.com/kubernetes/minikube>

Local Kubernetes tool that runs a single-node or multi-node cluster inside a VM or container, aimed at development and learning.

## Kubeone

- Official: <https://docs.kubermatic.com/kubeone/>
- GitHub: <https://github.com/kubermatic/kubeone>

Kubermatic's open source tool for installing, managing, and upgrading highly available Kubernetes clusters on any infrastructure.

## MicroK8s

- Official: <https://microk8s.io/>
- GitHub: <https://github.com/canonical/microk8s>

Canonical's lightweight, single-package Kubernetes distribution for workstations, edge, IoT, and CI, with easy add-ons.

## eksctl

- Official: <https://eksctl.io/>
- GitHub: <https://github.com/eksctl-io/eksctl>

The official CLI for Amazon EKS, used to create and manage clusters, nodegroups, and add-ons via a single declarative tool.

## Kind

- Official: <https://kind.sigs.k8s.io/>
- GitHub: <https://github.com/kubernetes-sigs/kind>

Kubernetes IN Docker: runs local Kubernetes clusters using Docker containers as nodes, primarily for testing Kubernetes itself and CI.
